When it starts a new job, Jenkins SGE Plugin tries to use an existing idle agent. If there are no idle agents, a new agent is automatically provisioned.

When things are less busy, idle agents are terminated after being idle for the Maximum idle time specified here.

Each project has a Workspace button that you can use to view the project workspace files in your web browser. This handy feature relies on the agent that executed the job. If the agent is gone, the workspace can no longer be viewed via the Jenkins web interface. If this happens often, consider increasing the Maximum idle time.

Note that the Maximum idle time applies only to newly provisioned agents. Existing agents retain the maximum idle time that was in effect when they were provisioned.

Handy conversions:

  • 1440 minutes = 1 day
  • 2880 minutes = 2 days
  • 4320 minutes = 3 days
  • 5760 minutes = 4 days
